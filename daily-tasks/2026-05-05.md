## Notes / thoughts

- Keep the outbox write in the same transaction as the mission completion insert.
  Even if the worker isn't processing yet, the write path must be atomic from
  day one — retrofitting that later is painful.
- Don't model the full reward type hierarchy today. A flat `RewardGrantIntent`
  record with a `RewardType` enum is fine for now. The ledger append is the
  architectural signal, not the reward catalog.
- Optimistic concurrency on player profile (ETag / row version) can be a stub
  today. Raise the concurrency conflict in domain but don't wire the HTTP `409`
  response until tomorrow.
- Dapper only goes in `MissionCompletionRepository`. Everything else uses EF
  Core. Don't get distracted generalising the repository pattern.

---

## Blockers / risks

- Schema decisions can drag if I overthink column names now. Pick something
  sensible and move on; migrations make it changeable.
- Integration tests need a live SQL Server container. Make sure Docker is running
  before the test run, or the integration suite will fail silently.
- `TreatWarningsAsErrors` will bite me if I leave any nullable reference warnings
  in new types. Fix them as I go, not in a cleanup pass at the end.
